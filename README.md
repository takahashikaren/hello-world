# ソフトウェア工学 第1回～第6回まとめ

## 第1回 ガイダンス

ソフトウェア工学は、品質の高いソフトウェアを効率的に開発・運用するための知識や技術を体系的に学ぶ学問である。講義では、ソフトウェア開発に関する知見を学ぶだけでなく、それを実践に結び付けることの重要性について説明された。また、ソフトウェア工学は個人の知識だけでは不十分であり、チーム全体で知識やルールを共有することが重要であることを学んだ。近年ではAIによるソフトウェア開発が進展しているが、その場合でも要件定義やテスト、バージョン管理などのソフトウェア工学の知識が必要であることを理解した。

## 第2回 ソフトウェア工学概論（1）

ソフトウェア工学では、単にプログラムを書くのではなく、開発プロセス全体を管理することが重要である。システム開発では要求分析、設計、実装、テスト、保守といった工程を経てソフトウェアが作られる。各工程で適切な成果物を作成し、品質を確保しながら開発を進める必要があることを学んだ。

## 第3回 ソフトウェア工学概論（2）

ソフトウェア開発では品質、コスト、納期のバランスが重要である。これらはQCD（Quality、Cost、Delivery）と呼ばれ、プロジェクトの目的によって優先順位が異なる。品質を重視するシステム、コストを重視するシステム、納期を重視するシステムが存在し、状況に応じた判断が求められることを学んだ。また、ソフトウェアの価値や規模を測定するために、コード量やファンクションポイント法などの評価方法が用いられることを理解した。

## 第4回 プロジェクトマネジメント（1）

プロジェクトとは、有期性と独自性を持つ活動であり、ソフトウェア開発もその一例である。プロジェクトを成功させるためには、目標から逆算して計画を立てるバックキャスティングの考え方が重要であることを学んだ。また、目標を具体的に設定し、達成に向けた筋道を考えながら進めることで、効率的なプロジェクト運営が可能になることを理解した。

## 第5回 プロジェクトマネジメント（2）

ソフトウェア開発にはさまざまな開発プロセスが存在する。ウォーターフォール型開発は進捗管理がしやすく成果物が明確である一方、後工程に負担が集中する可能性がある。アジャイル開発は変化への対応を重視し、短いサイクルでソフトウェアを提供する手法である。また、反復型開発やスパイラルモデルなども存在し、それぞれにメリットとデメリットがあるため、プロジェクトの特性に応じて適切な手法を選択することが重要であることを学んだ。

## 第6回 WBS

WBS（Work Breakdown Structure）は、プロジェクトを達成するために必要な作業を階層的に分解して整理する手法である。WBSを作成することで、プロジェクトのスコープが明確になり、必要な作業や役割分担を把握しやすくなる。作成時にはトップダウンアプローチとバックキャスティングを用いて作業を洗い出し、MECE（漏れなく・重複なく）の考え方で整理することが重要である。WBSはプロジェクト管理の基礎となる重要なツールであり、計画的な開発を支援することを学んだ。
